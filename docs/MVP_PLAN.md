# FachkundePilot — MVP Plan

## Phase 1: Experience Prototype Layer
- Build public landing page structure.
- Build certificate listing cards and detail placeholders.
- Build dashboard mock UX (static/demo data).
- Build learning page mock (section + lesson shell).
- Build flashcard interaction mock.
- Build exam simulation mock (UI-only flow).

## Phase 2: Core Data Backbone
- Implement Supabase schema and migrations.
- Implement auth + protected routing.
- Build admin content creation flows.
- Convert certificate pages to dynamic data-driven rendering.
- Convert lessons to dynamic content.
- Convert flashcards to dynamic content.

## Phase 3: Learning Engine Essentials
- Implement mock exam engine (question pull, scoring, result breakdown).
- Implement progress tracking model and dashboard metrics.
- Implement multilingual content rendering workflow.
- Implement glossary integration with lesson and flashcard contexts.

## Phase 4: Commercial & Intelligence Expansion
- Payment integration (Stripe).
- Subscription plans + entitlement checks.
- AI tutor assistant for explanations and practice guidance.
- Advanced analytics for learner outcomes and content quality.

## MVP Definition of Done
- At least 3 certificates fully usable end-to-end.
- Full lesson/flashcard/mock exam loop operational.
- Multilingual explanation support in DE/AR/EN/TR.
- Admin can create and publish content without developer intervention.
