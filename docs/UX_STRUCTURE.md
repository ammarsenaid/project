# FachkundePilot — UX Structure

## 1) End-to-End User Journey
1. Discover platform (SEO/content/referral).
2. Browse certificates and compare outcomes.
3. Register/login and choose certificate(s).
4. Complete onboarding (language preferences, goal date).
5. Start structured learning path (sections → lessons).
6. Reinforce with flashcards and glossary.
7. Simulate real exam via mock tests.
8. Review weak areas and continue iteration.
9. Reach target readiness and maintain retention.

## 2) Onboarding Flow
- Step 1: Welcome + value proposition.
- Step 2: Select target certificate(s).
- Step 3: Select preferred explanation language(s) (DE, AR, EN, TR).
- Step 4: Set weekly study goal and exam date.
- Step 5: Auto-generate first study plan and route to dashboard.

## 3) Dashboard Flow
- Snapshot cards:
  - Overall progress per certificate.
  - Current streak / weekly goal.
  - Next recommended lesson.
  - Latest mock exam score.
- Action rail:
  - Continue learning.
  - Start flashcard session.
  - Start mock exam.
- Weak-topic highlights and quick jump links.

## 4) Certificate Selection Flow
- Browse all certificate cards with category filters.
- Open certificate detail (overview, outcomes, syllabus structure).
- Enroll with one click.
- Land in certificate learning hub.

## 5) Lesson Learning Flow
1. Enter section page.
2. Open lesson (German source content first).
3. Toggle simplified explanation language.
4. Review key terms panel.
5. Complete lesson checkpoint.
6. Mark lesson complete and unlock next recommended item.

## 6) Flashcard Flow
- Start session by section or mixed mode.
- Card front: German term / concept prompt.
- Card back: explanation + optional translation + example.
- Self-rating (Easy/Medium/Hard) to schedule reviews.
- Session summary: accuracy, difficult cards, next review time.

## 7) Mock Exam Flow
- Choose exam mode (timed / practice).
- Question navigation panel (status colors: unanswered/reviewed/flagged).
- Submit and receive score + topic breakdown.
- Detailed review with correct answers and explanations.
- Suggested next lessons for weak areas.

## 8) Progress Tracking Flow
- Track by:
  - Certificate
  - Section
  - Lesson completion
  - Flashcard mastery
  - Mock exam performance
- Readiness indicator with transparent formula (not black-box).
- Timeline visualization for consistency.

## 9) Mobile User Flow
- Thumb-friendly bottom navigation in app areas.
- Prioritize “Continue” CTA and short sessions.
- Single-column content hierarchy.
- Sticky lesson controls and fast language toggles.
- Optimized mock exam interactions for small screens.

## 10) Accessibility Requirements
- WCAG 2.1 AA target.
- Minimum contrast compliance.
- Full keyboard navigation for core interactions.
- Screen-reader friendly labels and landmarks.
- Support for right-to-left rendering nuances where relevant (e.g., Arabic explanations).
- Motion reduction preference respected.
